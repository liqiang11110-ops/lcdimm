目标

板型 -> 每条用料 -> 每颗料数量 -> 替代料 -> 替代数量关系 -> 影响 -> 验证 -> 风险/状态

日常使用顺序

  • 先看硬件模块：判断物料属于 DRAM、PMIC、SPD Hub、CKD、阻容、PCB 哪一类。
  • 再看用量/替代料：确认每条内存用了几颗料，以及替代数量关系。
  • 录入新板型：填写板型、应用类型、容量、Rank、颗粒组织、颗粒数量、阻容数量、焊点数、风险。
  • 后续导入 BOM：每颗料都要落到实际用量，替代料必须写影响和验证项。

替代料必须写清楚

原料：原供应商 / 原料号 / 原描述 / 每条用量
替代料：替代供应商 / 替代料号 / 替代描述 / 每条替代用量
数量关系：例如 1 pcs -> 1 pcs
影响：例如 SPD 读取、XMP/EXPO、温度读取、主板识别
验证：例如 SPD dump、MemTest、温升、主板兼容
状态：待验证 / 工程确认中 / 品质确认中 / 已批准 / 不通过
风险：低 / 中 / 中高 / 高

高风险规则

  • DRAM 替换：高风险，必须看容量、x8/x16、Rank、SPD、平台兼容。
  • PMIC 替换：高风险，必须看电压轨、上电时序、温升、老化。
  • SPD Hub 替换：中高风险，必须看烧录、SPD dump、XMP/EXPO、温度读取、主板识别。
  • CKD 替换：高风险，只在 CUDIMM/CSODIMM 里重点管理。
  • 阻容替换：不能只看数量，要按位号和功能位置确认。
